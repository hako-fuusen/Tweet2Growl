Tweet2Growl

起動前の手順（アクセスキーの取得と設定）

はじめに

Tweet2Growlでは、アクセスキーの取得工程をユーザー側で実施する形式を取っています。

これは、アクセスキー流出による（製作者に振りかかる）諸問題を回避するためです。気になる方は「Twitter アクセスキー 流出」といった単語でググってください。

Twitterアプリを開発したことがある人向けの説明

Twitter Developersで新規アプリを登録して、API key, API secret, Access token, Access token secretを取得して、twitter4j.propertiesにそれぞれ設定してください。

アクセスキーの取得

Twitterアカウントを取得済みであることを前提に記載します。

Twitter developers にログイン

Twitter Developers にアクセスし、右上の Sign inからログイン。
ユーザー名とパスワードはTwitterのものと同じ。
右上のSign inが自分のアイコンになっていればログイン成功。

My applications に移動

右上に表示されているアイコンをクリックして表示表示されるメニューから My applicationsを選択。切り替わったページにCreate New Appと書いてあるボタンがあるのでそれをクリック。

Create New App

アプリケーションの情報を入力する画面になるので以下を参考に入力する。

  • Name
    • アプリケーション名を入力する欄だが、基本的には何でもOK。既に存在する名前だとエラーになるので適当に長い名前をつける。
  • Description
    • アプリケーションの説明文を入力する欄だが、こちらも基本的には何でもOK。10文字以上入力する必要があるのでこちらも適当に入力する。
  • Website
    • アプリケーションの公式ページのURLを入力する欄。 自分のツイッターのプロフィールページのURLあたりを を入力すればOK。
  • Callback URL
    • 使用しないので空欄のままでOK

Organization とか Application settings とか書かれたページが出れば登録完了。

アクセスキーの取得

Organization とか Application settings とか書かれたページのAPI Keysタブをクリック。
切り替わった先のページCreate my access tokenボタンをクリック。
少し待ってからRefreshのリンクをクリック。

Your access token以下にAccess tokenやAccess token secret が表示されていればアクセスキー取得成功なのでそのままブラウザを開いておく。表示されてない場合は更に少し待ってリロードする。

アクセスキーの設定

twitter4j.propertiesをテキストエディタで開く。
それぞれ以下のように編集して保存する。

  • oauth.consumerKey={API key}
    • {API key}の部分を、[API key]テキストボックスの文字列に置き換える
  • oauth.consumerSecret={API secret}
    • {API secret}の部分を、[API secret]テキストボックスの文字列に置き換える
  • oauth.accessToken={Access token}
    • {Access token}の部分を、[Access token]テキストボックスの文字列に置き換える
  • oauth.accessTokenSecret={Access token secret}
    • {Access token secret}の部分を[Access token secret]テキストボックスの文字列に置き換える

アクセスキー設定確認

付属の check_authorize.bat をダブルクリックして実行。
Twitterの認証に成功しました。という文言と、ユーザー名が表示されていれば設定完了。
